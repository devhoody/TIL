# System.out.print() vs BufferedWriter.write()

> 코딩테스트 중 print()와 write() 둘 중, 무엇을 써야하나 고민이 되어 두 메소드의 `메모리 사용`과 `처리 시간`에 차이를 알아보았다.

## 1. 메모리 사용

> 데이터 양이 많을 때, write()가 print()보다 성능이 좋다.

print()는 데이터를 콘솔에 직접 입력한다. 즉, 데이터를 메모리에 저장하지 않는다.

write()는 데이터를 파일에 쓰기 전에 버퍼에 저장한다. 버퍼의 크기가 넘는 데이터를 쓰게되면 버퍼는 flush되고 데이터가 파일에 기록된다. 버퍼의 기본 크기는 `8192바이트`이다.

## 2.시간

> 데이터 양이 많을 때, write()가 print()보다 성능이 좋다.

print()는 메모리 할당 및 추가 I/O작업을 수행할 필요가 있다.

반면, `write()는` 메모리에 저장된 버퍼에 데이터를 쓴 다음 버퍼를 파일에 쓰기 떄문에 `필요한 I/O작업의 수가 print()보다 적다.`

## 3.결론

메모리 사용량, 시간 모두

> 소량의 데이터 : print(), 대량의 데이터: write()
