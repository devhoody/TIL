# 캡슐화

## 캡슐화란?

캡슐화는 데이터 구조와 함수를 하나의 영역에 함께 정의하는 것을 뜻한다.
즉, `데이터 구조 + 함수`를 말한다.

이를 통해 `클래스의 내부 정보를 외부로부터 접근을 제어`할 수 있게 된다.

## 캡슐화의 장점

### 데이터 숨기기

- 클래스의 내부 데이터를 캡슐화함으로써 외부로부터 숨길 수 있다.
- 이로써 클래스 이용자가 공용 인터페이스 사용을 가능하게하는 `추상화 수준을 제공`한다.

### 코드 구성 및 유지 용이성

- 복잡한 코드 및 함수와 메서드를 분류하여 `전체 시스템을 보다 체계적으로 이해`하고 개발 및 유지 관리를 하기 쉽게 만든다.

### 코드 재사용

- 캡슐화를 사용하면 `클래스를 사용하는 외부 코드에 영향을 주지 않고 내부 구현을 수정`할 수 있다. 이는 코드의 재사용을 가능하게 한다.

> 이클립스에서의 코드 재사용 방법은 [해당 페이지]()에 따로 정리되어있다.
