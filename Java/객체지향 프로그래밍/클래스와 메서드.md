# 클래스와 메서드

> 자바의 정석 3판과 유튜브 강의를 듣고 정리한다.

## 클래스의 또 다른 정의

- 변수 : 하나의 데이터를 저장할 수 있는 공간
- 배열 : `같은 타입`의 여러 데이터를 저장할 수 있는 공간
- 구조체 : `다른 타입`의 여러 데이터를 저장할 수 있는 공간
- 클래스 : 데이터와 함수의 결합 (`구조체 + 함수`)

클래스는 `사용자 정의 타입`이라고도 불린다.

## 코드 - 비객체지향 vs 객체지향

- 객체지향

```java
int hour = 12;
int minute = 34;
int second = 56;
```

- 비객체지향

```java
Time t = new Time();

t.hour = 12;
t.minute = 34;
t.second = 56;
```

## 3.1 선언위치에 따른 변수의 종류

### 영역의 종류

- 클래스 영역
  - 선언만 가능
  - iv, cv
- 메서드 영역
  - lv

### 변수의 종류

- 인스턴스변수
  - 인스턴스 만들 때 생성
- 클래스 변수
  - 클래스가 메모리에 올라갈 때 생성
- 지역변수
  - 메서드 내에서 변수를 선언할 때 생성

```java
class Variable {
	int iv; //인스턴스 변수
	static int cv; //클래스 변수

	void method() {
		int lv = 0; //지역 변수
	}
}
```

SSD나 HDD에 있는 데이터가 메모리로 이동(로딩) -> 메모리에서 CPU로 이동.

## 3.2 클래스변수와 인스턴스 변수

인스턴스 변수는 인스턴스가 생성될 때마다 생성되므로 `인스턴스마다 각기 다른 값`을 가질 수 있지만, 클래스 변수의 경우, 모든 인스턴스 변수가 `하나의 저장공간`을 공유하므로 `항상 공통된 값`을 갖는다.

카드를 예로 들어보자.

- 카드의 크기나 폭은 모든 카드에 똑같이 적용된다. => **_클래스 변수_**
- 카드의 번호나 무늬는 각 카드마다 개별적으로 다르다. => **_인스턴스 변수_**

```java
class Card {
	// 인스턴스 변수
	String kind; // 무늬
	int number; // 숫자

	// 클래스 변수
	static int width = 100; // 폭
	static int height = 300; // 높이
}
```

## 3.3 메서드(method)

- 메서드란, 작업을 수행하기 위한 명령문의 집합
- 메서드의 장점
  - 중복 제거
  - 관리 용이
  - 코드 재사용
- 정의 방법
  - 리턴 타입 있는경우 : `리턴타입` 메서드이름(타입 변수명)
    - 리턴 타입이 있는 경우엔 반드시 `return문`을 넣어주어야한다.
  - 리턴 타입 없는 경우 : `void` 메서드이름(타입 변수명)

## 3.4 Return문

- 메서드가 정상으로 종료되는 경우
  - 메서드의 {}에 도달했을 때
  - 메서드 수행 중 return문을 만났을 때

## 3.5 메서드의 호출

- 메서드의 호출 방법

```java
public static void Main(String[] args) {
	Math m = new Math(); // 참조변수 m 선언

	long result = m.min(5L, 3L); // 메서드 호출 : 참조변수.메서드이름(매개변수)
}

class Math{
	// 메서드 선언
	long min(long a, long b) {
		long result = 0;
		if (a > b) {
			result = b;
		} else
			result = a;
		return result;
	}
}
```
