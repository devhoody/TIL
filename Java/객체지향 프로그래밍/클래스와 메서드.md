# 클래스와 메서드

> 자바의 정석 3판과 뉴렉처 강의, 김영한님의 인프런 강의를 듣고 정리한다.

## 클래스가 필요한 이유
- 학생 이름과 나이, 점수로 구성된 데이터를 관리한다고 하자. 배열을 이용해서 관리할 수 있지만, `배열을 사용한다면 추후 관리하기가 까다롭다. 왜냐면, 만약 특정 학생의 데이터를 지우고 싶을 때, 그 학생에 해당하는 index의 데이터를 수동으로 작업해야하는 번거로움이 존재한다.` 이를 해결하기위해서 학생 자체의 개념을 만들어 활용하면 학생만 지우면 동시에 그에 해당하는 나이, 점수가 제거되므로 훨씬 관리가 수월하다. 개념을 이용하는 것이 곧 클래스를 이용한다고 볼 수 있다.

```java
public class Student {
    String name;
    int age;
    int grade;

}
```
## 참조값을 변수에 보관 해야 하는 이유

- 객체(인스턴스)를 생성하는 `new Student()`는 메모리에 객체를 생성하는 코드로 따로 이름이 없다. 이름이 없으므로 메모리에 있는 객체에 접근할 방법이 없다. 이를 위해서 변수에 참조값(메모리 주소)를 저장하여 변수를 이용하여 메모리에 있는 객체에 접근이 가능해진다.
- 참조값(메모리주소) 출력
    - `class1.Student@13221655`

## 클래스

- 변수 : 하나의 데이터를 저장할 수 있는 공간
- 배열 : `같은 타입`의 여러 데이터를 저장할 수 있는 공간
- 구조체 : `다른 타입`의 여러 데이터를 저장할 수 있는 공간
- 클래스 : 데이터와 함수의 결합 (`구조체 + 함수`)
- 멤버변수 : 클래스에 정의한 변수

클래스는 `사용자 정의 타입`이라고도 불린다.

## 코드 - 비객체지향 vs 객체지향

- 비객체지향

```java
int hour = 12;
int minute = 34;
int second = 56;
```

- 객체지향

```java
Time t = new Time();

t.hour = 12;
t.minute = 34;
t.second = 56;
```

## 3.1 선언위치에 따른 변수의 종류

### 영역의 종류

- 클래스 영역
  - 선언만 가능
  - iv, cv
- 메서드 영역
  - lv

### 변수의 종류
- 멤버변수 : 클래스에 정의한 변수
  
- 인스턴스 변수
  - 인스턴스 만들 때 생성
- 클래스 변수
  - 클래스가 메모리에 올라갈 때 생성
- 지역변수
  - 메서드 내에서 변수를 선언할 때 생성
  - 

```java
class Variable {
	int iv; //인스턴스 변수
	static int cv; //클래스 변수

	void method() {
		int lv = 0; //지역 변수
	}
}
```

## 객체 vs 인스턴스

- 객체는 `클래스에서 정의한 속성과 기능을 갖고있는 실체`
- 인스턴스는 `특정 클래스에서 생성된 객체`, 즉, `객체가 어떤 클래스에 속해있는지`에 초점을 맞출때 사용하는 용어

## 3.2 클래스 변수와 인스턴스 변수

인스턴스 변수는 인스턴스가 생성될 때마다 생성되므로 `인스턴스마다 각기 다른 값`을 가질 수 있지만, 클래스 변수의 경우, 모든 인스턴스 변수가 `하나의 저장공간`을 공유하므로 `항상 공통된 값`을 갖는다.

카드를 예로 들어보자.

- 카드의 크기나 폭은 모든 카드에 똑같이 적용된다. => **_클래스 변수_**
- 카드의 번호나 무늬는 각 카드마다 개별적으로 다르다. => **_인스턴스 변수_**

```javax`
class Card {
	// 인스턴스 변수
	String kind; // 무늬
	int number; // 숫자

	// 클래스 변수
	static int width = 100; // 폭
	static int height = 300; // 높이
}
```

## 3.3 메서드(method)

- 메서드란, 작업을 수행하기 위한 명령문의 집합
- 메서드의 장점
  - 중복 제거
  - 관리 용이
  - 코드 재사용
- 정의 방법
  - 리턴 타입 있는경우 : `리턴타입` 메서드이름(타입 변수명)
    - 리턴 타입이 있는 경우엔 반드시 `return문`을 넣어주어야한다.
  - 리턴 타입 없는 경우 : `void` 메서드이름(타입 변수명)

## 3.4 Return문

- 메서드가 정상으로 종료되는 경우
  - 메서드의 {}에 도달했을 때
  - 메서드 수행 중 return문을 만났을 때

## 3.5 메서드의 호출

- 메서드의 호출 방법

```java
public static void Main(String[] args) {
	Math m = new Math(); // 참조변수 m 선언

	long result = m.min(5L, 3L); // 메서드 호출 : 참조변수.메서드이름(매개변수)
}

class Math{
	// 메서드 선언
	long min(long a, long b) {
		long result = 0;
		if (a > b) {
			result = b;
		} else
			result = a;
		return result;
	}
}
```

# Static 메서드와 Instance 메서드

> iv(인스턴스변수) 사용 여부에 따라 두 메서드의 사용 여부가 정해진다.

### 인스턴스 메서드

- 인스턴스 생성 후, `참조변수.메서드이름()`으로 호출
- `iv, im과 관련된` 작업을 하는 메서드
- 메서드 내에서 `iv 사용가능`

### Static 메서드

- 객체생성없이 `클래스이름.메서드이름()`으로 호출
- `iv, im과 관련없이` 작업을 하는 메서드
- 메서드 내에서 `iv 사용불가`

```java
class MyMath {
	long a, b; // iv

	long add() { // instance 메서드, 이때, a,b는 iv
		return a + b;
	}

	static long add(long a, long b) { // static 메서드 , long a와 long b는 지역변수(lv)
		return a + b;
	}
}

class MyMathTest {
	public static void main(String[] args) {
		System.out.println(MyMath.add(200L, 300L)); // static메서드 호출, 객체생성없이 가능

		MyMath mm = new MyMath(); // 객체생성
		mm.a = 200L;
		mm.b = 300L;
		System.out.println(mm.add()); // instance 메서드 호출
	}
}
```

## 멤버간의 참조와 호출

같은 클래스의 멤버간에는 객체 생성이나 참조변수 없이 참조할 수 있다. 하지만, `static멤버는 instance멤버를 참조할 수 없다.`

### 메서드의 호출

- static메서드에서는 instance메서드를 호출할 수 없다.
  - 왜? => instance메서드는 `iv로 작업하는 메서드로 객체 생성이 필요`하다. 하지만 static 메서드에서 호출 할 때에는 `객체가 생성되어 있는지 확실하지 않기 때문`이다.

```java
class TestClass1 {
	void instanceMethod() {}  // 인스턴스 메서드
	static void staticMethod() {} // static 메서드

	void instanceMethod2() {  // 인스턴스 메서드
		instanceMethod(); // 다른 인스턴스 메서드 호출
		staticMethod(); // 다른 static 메서드 호출
	}

	static void staticMethod2 () { // static 메서드
		instanceMethod(); // instance 메서드 호출 에러발생!
		staticMethod(); // static 메서드 호출
	}
}
```

### 변수의 접근

```java
class TestClass2 {
	int iv; // instance 변수
	static int cv; // 클래스 변수(static 변수)

	void instanceMethod() { // 인스턴스 메서드
		System.out.println(iv); // instance 변수 사용
		System.out.println(cv); // class 변수(static 변수)
	}

	static void staticMethod() { // static 메서드
		System.out.println(iv); // instance 변수 사용 시 에러 발생!
		System.out.println(cv); // class 변수(static 변수)

	}
}
```
