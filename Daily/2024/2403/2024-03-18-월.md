# CS

## 컴파일

- 컴파일이란?
    - 개발자가 작성하는 소스코드를 컴퓨터가 이해할 수 있는 기계어로 번역시켜주는 과정
- 컴파일을 사용하는 이유
    - 사람이 기계어로 코드를 작성한다면, 매우 불편하고 서로 소통하기 힘들다. 그래서 우리가 사용하는 언어(영어)로 코드를 작성하여 보다 쉽게 코드를 작성할 수 있다.
- 컴파일의 원리
    - **전처리**
        - 실행되어야할 파일을 선정한다.
    - **컴파일**
        - 소스코드 → 어셈블리어(바이트코드)
    - **어셈블리**
        - 어셈블리어 → 기계어(바이너리코드, 0과 1로 이루어져있음)
    - **링크**
        - 만약 하나의 실행 코드에 여러 파일이 필요하다면, 각각 파일의 기계어를 합쳐주는 기능
- 특징
    - 자바 : java 파일 → class 파일
    - class파일은 자바 가상 머신(JVM)이 이해할 수 있는 코드 → 컴퓨터가 이해할 수 있는 코드x

# 자바 메모리 구조

## 메서드, 스택, 힙 영역

- 개념
    - 메서드 영역
        - 프로그램 실행시 공유되는  영역
        - `클래스 정보(실행 코드)`, `static` 영역, `상수` 풀 등
    - 스택 영역
        - 실행되고 있는 메서드 영역
        - 자바 실행 시, 하나의 실행 스택 프레임이 생성된다.
            - 스택 프레임 : 스택 영역에 쌓이는 1개의 공간, 메소드 호출할때마다 1개씩 생성되고, 메소드가 종료되면 해당 스택 프레임은 제거된다.
            - 1쓰레드 - 1 스택 프레임
        - **`지역 변수`**
    - 힙 영역
        - 객체(인스턴스)가 생성되는 영역, new 명령어 → `인스턴스`, `배열`

### 클래스에 존재하는 메서드 참조

- 만약, 특정 클래스의 인스턴스를 100개 이상 생성한다면 힙 영역에 인스턴스가 100개 이상이 생성된다.
- 변수는 인스턴스마다 다르다. 반면에, 메서드는 메서드 영역을 참조하여 공유한다.
    - 인스턴스에 각각 메서드가 존재한다면, 메모리 누수가 발생할 가능성이 있다.
- 인스턴스의 메서드를 호출하면 메서드 영역에 있는 코드를 불러서 호출한다.

## 스택 영역

- 자바는 스택 영역을 이용하여 메서드 호출과 지역 변수를 관리한다.
- 스택 영역은 FIFO이다.
    - FIFO : 먼저 넣은 것을 먼저 뺀다. → 위가 뚤린 박스
- 스택 프레임이 제거되면 자바 프로그램은 종료된다.

### 스택과 큐

- 스택
    - FIFO
- 큐
    - LIFO
    

### FIFO vs LIFO

- FIFO(First In First Out, 후입 선출)
    - 가장 최근의 것을 먼저 사용한다.
    - 예시) 위가 뚤린 그릇의 맨 위에 것부터 꺼낸다.
- LIFO(Last In First Out, 선입 선출)
    - 가장 먼저 들어간 것을 먼저 사용한다.
    - 예시) 계산대에 있는 고객들을 순서대로 처리한다.
