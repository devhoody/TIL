# 논리 설계

- E와 R에 해당하는 관계를 `테이블`로 정의
- 추상적인 데이터 구조가 보여지는 개념설계를 바탕으로 Key, 속성, 관계등을 표시하여 보다 구체화된 데이터로 작성한다.


## 키 엔티티 , 액션 엔티티

- E : Entity에 해당하는 테이블을 먼저 그린다. - `키 엔티티`
- R : Entity사이의 관계를 그린다. - `액션 엔티티`
    - **1대 N** : N쪽의 테이블로 1의 속성이 들어간다.
    - **N대 N** : 새로운 테이블로 구성한다.

### 엔티티

- 정의 가능한 사물 또는 개념
- **데이터베이스의 테이블이 엔티티로 표현**된다.

### 엔티티 속성(Attribute)

- 엔티티에는 해당 엔티티를 구성하는 속성들을 포함한다.
- **데이터베이스의 테이블 내 각 칼럼들**을 속성이라고 보면 된다.

### 엔티티 도메인(Domain)

- 각 속성에 대한 값, 타입, 제약사항 등을 표현하는 것
- 사용자에 따라 속성 타입만 그릴 수 있고, 생략할 수도 있다.

## 식별자

### 주 식별자(Primary Key, PK)

- 데이터베이스 테이블의 Primary Key
- **중복이 없고, NULL값을 갖지 않는 값**
- 다른 속성과 확실히 구분되어야 한다.

### 외래 식별자(Foreign Key, FK)

- 데이터베이스 테이블의 Foreign Key
- 외래 식별자를 사용할 때는 **선을 이어주는데 개체와의 관계를 따져 표시**한다.

### 기본키/ 후보키/ 대체키/ 대리키/ 복합키/ 외래키

- 기본키
    - 해당 테이블의 기본 PK키
- 후보키
    - 해당 테이블의 속성 중 식별키가 될 수 있는 키들
- 대체키
    - 기본키가 될 수 있는 후보키 중에서 탈락한 키들
- 대리키
- 복합키
    - 특정 데이터를 식별할 필요가 없는 도메인
    - *ex) 걸음걸이 측정, 좋아요→ 게시글 + 회원 아이디*
        
- 외래키

## 식별자 vs 비식별자

- 식별자 관계
    - 부모와 자식 관계에서 자식의 테이블이 **부모 테이블의 PK**를 참조하여 **`자신의 PK`**로 이용하는 관계
    - `실선`으로 연결
        
- 비식별자 관계
    - 부모와 자식 관계에서 자식의 테이블이 **부모 테이블의 PK**를 참조하여  `**자신의 일반속성**`로 이용하는 관계
    - `점선`으로 연결
