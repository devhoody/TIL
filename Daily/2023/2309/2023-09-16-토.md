# 인프라 아키텍처

- 인프라란?
    - 사용자가 서비스를 이용하기위해 필요한 **기반**
- 인프라 아키텍처란?
    - 필요한 **기반을 구성하는 구조**

## 물리서버와 논리서버

- 서버란?
    - 아키텍처에서 이용되는 컴퓨터
- 물리서버
    - **컴퓨터 자체**를 가리키는 서버
- 논리서버
    - **컴퓨터 안**에서 역할을 갖는 서버
    - 예) 웹서버, DB서버

## 집약형과 분할형 아키텍처

- 집약형 아키텍처
    - 사용자 입출력, 데이터 입출력 등 모든 기능이 **하나의 대형컴퓨터**가 처리하는 구조
- 분할형 아키텍처
    - **여러대의 컴퓨터**를 이용하여 시스템을 구성하는 구조

## 수직 분할형 아키텍처

- 서버(컴퓨터)별로 **다른 역할을 담당**하는 분할형 아키텍처

### 클라이언트-서버형 아키텍처

- 클라이언트와 서버로 나누어 클라이언트에서는 간단한 처리를 하지만, **필요한 경우 서버에서 모든 처리를 담당**하는 아키텍처
- 단점
    - 클라이언트 측에서 **정기적인 업데이트**가 필요하다.

### 3계층형 아키텍처

- 대부분의 시스템을 이루는 구조로 클라이언트측에서의 정기적인 업데이트를 보완할 수 있는 아키텍처
- `프레젠테이션, 애플리케이션, 데이터 계층`으로 나눈다.
- 프레젠테이션 계층
    - 사용자의 입력과 화면 출력
- 애플리케이션 계층
    - 사용자 요청에 따라 업무 처리
- 데이터 계층
    - 애플리케이션에서 보낸 요청에 따라 데이터 입출력

## 수평 분할형 아키텍처

- 수직 분할형이 기능별 서버를 나누었다면, 수평 분할형은 **`같은 기능`을 가진 서버를 늘려나가는** 아키텍처

### 단순 수평 분할형 아키텍처

- 단순하게 같은 계층에서 서버를 늘린것으로 늘어난 서버끼리 `데이터를 공유할 수 없다.`
- 예) 서울지사의 데이터와 부산 지사의 데이터 (공유x)

### 공유형 아키텍처

- 같은 계층의 서버끼리 `데이터를 공유할 수 있는` 아키텍처
- 예) 서울지사와 부산 지사끼리 데이터 공유

## 지리 분할형 아키텍처

- 업무 연속성과 시스템 가용성을 높이기 위한 아키텍처

### 스탠바이형 아키텍처

- 물리 서버를 최소 2대 이상 준비하여 만약 하나의 물리 서버에서 문제가 발생했을때, 준비되있던 다른 물리 서버에서 실행하고있는 소프트웨어를 재시작하여 이어나가는 아키텍처
- **“소프트웨어를 재시작한다.”**= **`‘페일오버’`**

### 재해 대책형 아키텍처

- **서버 장비를 최소 구성 및 동시 구성**으로 별도 사이트에 배치하여 메인 환경에서 재해가 발생했을 시  대응할 수 있는 아키텍처
- 메인 서버와 재해 대비용의 서버끼리 `데이터를 항시 동기화`하여 만일에 대비한다.
