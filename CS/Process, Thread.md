# 프로세스와 스레드

### 프로세스란?

- 운영 체제로부터 할당받은 `작업 단위(메모리 공간을 얻음)`
- 프로그램이 돌아가고 있는 상태
- 프로세스만 존재할 때에는 하나의 작업만 수행이 가능해서 멀티태스킹을 하지 못했다. 이를 해결하기 위해 나타난것이 스레드이다.

### 스레드란?

- 프로세스가 할당받은 자원을 이용하는 `작업 흐름`
  - ex) 크롬 브라우저에서 파일 다운로드, 인터넷 쇼핑, 음악 감상 등 여러 기능을 동시에 수행하게 만들어줌.

### 프로세스 vs 스레드

- `CPU 부하정도` : 프로세스 > 스레드
  - 프로세스는 생성 시 독자적인 메모리 공간을 가지기에 스레드와 비교해 CPU부하 정도가 높다.
- 프로세스의 장점 : `개별 처리 독립성`이 높다는 것이다.
- 스레드의 단점 : 메모리 공간을 공유하기때문에 `의도하지 않은 데이터 읽기/쓰기 발생할 수 있다.`
- 단, 프로세스가 메모리 공간을 공유할 수 없는 것은 아니다. 개발자에 따라 하기 나름!
