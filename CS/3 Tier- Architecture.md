# OS커널

- OS에서 가장 중요한 핵심!! `인터페이스`를 제공한다!!

## OS커널의 6가지 기능

1. `시스템 콜 인터페이스` : 프로세스나 스레드로부터 명령을 받는 인터페이스
2. `프로세스 관리` : 가동되고 있는 프로세스 관리와 CPU 이용 우선순위 관리
3. `메모리 관리` : 서버상의 메모리를 단위 크기 블록으로 분할해서 프로세스에 할당
4. `네트워크 스택` : 네트워크 관리
5. `파일 시스템 관리` : 파일 시스템을 관리
6. `장치 드라이버` : 디스크, NIC, HBA 등의 물리 장치와 작업

### 시스템 콜 인터페이스

- 애플리케이션이 OS통해 어떤 처리를 하고자 할때, ‘`시스템콜`’이라는 명령을 이용해서 `커널에 명령을 내림`. 이때 명령은 `인터페이스를 통해 전달`된다. ex)은행의 접수 창구
  _ex) 데이터 읽기 쓰기_

### 프로세스 관리

- OS상에서는 수백 수천개의 프로세스 가동이 가능한데, CPU의 코어 수는 많아야 수십개이다. 그러므로 `CPU의 이용 우선순위를 정하는 것이 중요`하다. 이것이 커널의 핵심기능 중 하나!
  _ex) 인솔 선생님의 “줄 맞춰서세요.” 등_

### 메모리 관리

- 프로세스관리가 CPU 코어를 고려했다면, 메모리 관리는 `메모리 공간의 최대치를 고려`한다.
- 프로세스, 스레드가 할당받는 메모리 공간을 처리. 이 메모리 공간은 프로세스 및 스레드가 소유하고있는 **개인 공간**. `프로세스 시작시에 확보`된다.

### 파일 시스템 관리

- 파일시스템이란, `물리 디스크에 제공된 데이터를 관리하는 기능`
- 프로세스(스레드)는 편리성 때문에 모든 것을 파일 단위로 생각.
- 물리 디스크에 기록된 데이터는 10100으로 숫자 집합에 불과하기에 그대로 사용하기 어렵다. 파일 시스템 덕분에 파일 단위로 데이터 작성 / 삭제가 가능하다.

### 장치 드라이버

- 디스크나 NIC 등의 `물리 장치용 인터페이스`를 제공.

## 커널 설계 및 구현 방식 두가지 : 모놀리식커널과 마이크로 커널

### 모놀리식 커널

- 단일형으로 `OS의 모든 구성요소를 하나의 메모리 공간`을 통해 제공.
  _ex)한 명의 CEO가 모든 것을 제공하고 있는 형태_
  ex) 유닉스계열 OS, 리눅스

### 마이크로 커널

- `최소한의 기능만 커널이 제공하`고 그외 기능은 커널 밖에서 제공
- 커널 자체가 작아지기 때문에 모놀리식에 비해 심플하다.
  _ex) 맥 OS_

# 웹 데이터의 흐름

## 클라이언트 → 웹서버

- 전체적인 흐름
  - _웹 브라우저의 요청 → 이름 해석 → 웹 서버 요청접수 → 웹 서버가 동적 or 정적 콘텐츠 판단 → 필요한 경로로 데이터 엑세스_
- `HTTP 요청`에 의해 적절한 파일이나 콘텐츠를 반환

### 정적 콘텐츠란?

- 실시간으로 `변경할 필요가 없는` 데이터
  - ex) 회사 로고 데이터 등
- `데이터 갱신 빈도가 낮은 것`은 웹서버 내 디스크에 저장.

### 동적 콘텐츠란?

- `높은 빈도로 변경`되는 데이터
  - ex) 은행 잔고 등
- 이는 갱신 빈도가 높아 `디스크에 넣으면 병목현상의 원인`이 될 수 있다.
- AP 서버가 HTML 파일을 동적으로 생성한다.

## 웹 서버 → AP 서버

- 전체적인 흐름
  - _웹 서버로부터 요청이 도착 → 스레드가 요청 받으면 자신이 계산 가능 여부에 따라 DB 접속 여부 판단 → DB접속 필요시 연결 풀에 액세스 → DB서버에 요청을 던진다._
- AP 서버란, `‘동적 콘텐츠’ 요청에 대한 처리`를 하는 서버
- 자바 가상 머신인 `JVM`도 이에 속하고 하나의 거대한 프로세스로 그 안에 여러 스레드 중 하나가 요청을 접수한다.
- AP서버가 DB서버에 접속하려면 ‘드라이버’가 필요하다. 드라이버는 커널 장치와 비슷하게 데`이터베이스로 접속하는 인터페이스를 제공해 데이터베이스를 은닉화한다.`

### DB서버 이외의 옵션

- 데이터가 필요하면 DB서버에 접속하는 것이 맞지만 **항상 효율적이진 않다.**
- 예를 들어, 대한민국 행정 정보는 자주 바뀌는 것이 아니기에 매번 데이터베이스에 요청하는건 비효율적. 이처럼 `규모가 작고 갱신정도가 낮은 데이터`는 JVM내부에 `캐시로 저장해두었다가 반환하는게 좋다.`
- 반대로 `규모가 큰 정적 데이터 전송` 시에는 DB 서버 이외에 `CDN(Content Delivery Network)라 불리는 데이터 전송 전용 서버`를 이용하는 경우다 있다.

### CDN

- 대부분의 `웹시스템에서는 CDN을 이용하고있다.`
- 웹 시스템의 특징인 `‘하나의 시스템을 수많은 사용자가 이용한다.’` 라는 것에 기인.
- CDN은 대량의 데이터 전송에 특화된 것으로, 전 세계에 있는 데이터 복사본(캐시)을 배치하는 기술과 병렬 기술을 활용해서 처리를 효율화 하고 있음.
- _기업형 시스템에서는 데이터 갱신, 제한된 사용자 수로 인해 CDN을 적극적으로 사용하진 않는다._
